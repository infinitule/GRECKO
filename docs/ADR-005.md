# ADR-005: Comms as an injected environment, not an allocator assumption

**Status:** Accepted  
**Phase:** PC — Comms & degradation model

## Context

Every allocator in the plan (GreedyAuction / EconomicMDP / CBBA) must operate
on a comms graph that is injected at runtime, not hardcoded. The contested EM
environment is an input — degrading it must be a config change, not a code change.

## Decisions

### Abstract link model only

`LinkModel` is a pure predicate: given two node positions and current time, is
the link up? Implementation: distance gate → denial-zone geometry → duty-cycle
intermittency → Bernoulli drop. No RF physics, no waveform, no jammer design —
per scope boundary. "Jamming" here means "denial zone active"; the mechanism is
a radius and a circle, not an emitter model.

### Degradation modes are config, not code

Five YAML scenario files cover the acceptance test matrix:
`clear`, `denial_zone`, `range_collapse`, `intermittent`, `full_partition`.
Adding a new mode (e.g. shadow fading) means a new YAML key and one new predicate
clause in `link_up` — no downstream code changes.

### Denial zones block link-segment intersections, not just endpoints

A zone that an endpoint merely approaches but doesn't enter would still block
the RF path. `DenialZone.crosses_segment` implements the point-to-segment
distance check so the geometry is correct. CI tests verify this case.

### PartitionTracker / `topology()` publishes connected components every tick

`CommsNetwork.topology(t)` returns a `MeshTopology` message (edges, partition
list, count) that the allocator receives as its adjacency structure each round.
This is the single interface: allocators are forbidden to call `link_up` directly.

### Isolation: comms imports nothing from sensing/fusion/classify/core-entities

The comms layer is the substrate everything routes through. It must depend on
nothing above it — enforced by CI grep (test #16).

### Intermittency is desynchronised per pair

`_pair_phase(a, b)` gives a deterministic but pair-specific phase offset (SHA-256
of sorted ids) so the whole mesh doesn't blink simultaneously. This produces
realistic fragmentation patterns without any global clock.

## Rejected alternatives

- **RF-physics link budget:** requires emitter parameters, which are outside
  scope. Also produces false precision for a sim whose kinematic resolution is
  already an abstraction. Rejected.
- **Hard-coded comms graph per scenario:** couples scenario design to allocator
  code; makes parameterised sweeps impossible. Rejected.
- **Separate "comms world" that allocators call directly:** allocators would
  import comms internals, violating the layer isolation. Adjacency dict injected
  per round is the clean interface. Rejected.

## Acceptance evidence

- Denial-zone: zero messages cross or originate inside a zone (16 assertions).
- Partition count: hand-computed 3-cluster scenario matches (verified).
- Bridge merge: adding one mid-node collapses 2 partitions to 1 (verified).
- Full partition: N nodes → N partitions, zero edges (verified).
- Degradation sweep (30 seeds per radius, 12 interceptors, 12 targets):

| radius | coverage |
|---|---|
| 4000 m | 1.000 |
| 2000 m | 0.989 |
| 1000 m | 0.803 |
| 500 m | 0.653 |
| 100 m | 0.586 |

Coverage is monotone non-increasing with radius (test-asserted). The curve
reproduces cleanly with `python -m sim.comms.sweep`.
