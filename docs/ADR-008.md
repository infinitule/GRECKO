# ADR-008: P-S2R — sim-to-real validation strategy

**Status:** Accepted
**Phase:** PS — sim-to-real validation strategy

## Context

Every conclusion GRECKO produces is conditional on modelled parameters:
sensor detection curves, measurement noise, clutter rates, link loss, threat
kinematics. None of these have been measured against hardware. Before any
result crosses the reality gap, two questions must be answerable:

1. **Sensitivity** — which modelled parameters do the conclusions actually
   depend on, and how hard?
2. **Validation** — for each such parameter, what real-world measurement
   would pin the sim value to reality, and what happens if it fails?

P-S2R answers both with a machine-checked harness (`/s2r`) rather than a
prose-only strategy document, so the answers regress like everything else.

**Scope:** simulation and analysis only. The validation gates describe
measurement and data collection (bench, hardware-in-the-loop injection,
instrumented field-analog observation). No gate involves hardware control,
RF transmission design, or effector integration — and a test greps the gate
text for forbidden vocabulary to keep it that way.

## Decisions

### The reality gap is a bounded 8-dimensional perturbation envelope

`RealityGap` perturbs exactly the parameters a deployment cannot take on
faith: `pd_scale`, `noise_scale`, `clutter_scale`, `latency_add_s`,
`update_rate_scale`, `comms_drop_add`, `comms_radius_scale`,
`target_speed_scale`. Bounds are one-sided toward pessimism (real sensors
underperform spec; real threats fly faster than the library assumed).
Perturbation is applied by building modified *copies* of the sensor mesh and
comms config — `/sim` is never mutated, and the POSG invariant is untouched:
the gap changes how truth is observed, never who may read it.

### Engagement margin is the headline metric, not intercept count

Interceptors pursue in truth space once assigned, so intercept *count* is
nearly insensitive to sensing fidelity — a misleading robustness signal.
What sensing degradation actually costs is reaction time, which shows up as
intercepts happening closer to the asset. The probe metric is therefore
**margin**: the minimum intercept distance from the asset (threshold 100 m).
This is also the operationally meaningful quantity (standoff/debris margin).

### Fixed probe attack isolates the gap's effect

One fixed 5-UAS attack (4 main + 1 feint, 600 m, sized to the bridge's 3
consumable interceptors) is held constant across all studies. Any outcome
change is attributable to the perturbation, not red-team variation. The
sensitivity statistic is the minimum margin over a 3-seed set, so rankings
reflect worst-case behaviour rather than single-seed luck.

### One validation gate per gap dimension, machine-checked coverage

`s2r/gates.py` maps each dimension to exactly one `ValidationGate`:
parameter, tier (`bench` / `hil` / `field-analog`), measurement procedure,
machine-checkable accept criterion, and the sim conclusion at risk if the
gate fails. `coverage_check()` returns the uncovered dimensions and the
acceptance suite requires it to be empty — adding a gap dimension without a
gate breaks CI.

### Insensitive dimensions are limitations, not robustness

The OAT sweep found the sim does not respond at all to `latency_add_s`
(fusion timestamps reports by scan tick, so report latency is invisible),
`comms_drop_add`, and `comms_radius_scale` (all interceptors sit inside
comms range of C2 even at half radius in the bridge geometry). A zero delta
means the sim *cannot price* that parameter — so its gate must pass before
any margin conclusion crosses the gap. `SensitivityReport.insensitive_dims()`
surfaces these explicitly and a test requires each to carry a gate.

### Domain randomization quantifies conclusion stability

`robustness_study()` samples K gaps uniformly from the envelope and reports
the **stability fraction**: the share of the plausible reality envelope in
which the margin conclusion survives, plus the worst case and the failing
gap vectors. A conclusion that only holds at nominal fidelity is a
simulation artefact, not a result.

## Measured results (seeds 0–2, fixed probe)

- Nominal engagement margin: **237.7 m** (clears the 100 m threshold).
- Tornado ranking (margin lost at worst bound):
  `noise_scale` −133.7 m, `update_rate_scale` −96.1 m,
  `target_speed_scale` −76.6 m, `pd_scale` −40.4 m, `clutter_scale` −14.2 m;
  `latency_add_s` / `comms_drop_add` / `comms_radius_scale` insensitive.
- Validation priority: fund the measurement-scatter (bench) and sustained
  scan-rate (soak) gates first; the Pd field campaign third.
- Domain-randomized stability: **0.75** over 8 envelope samples
  (worst case 45.1 m margin — conclusion does NOT survive the full envelope,
  which is the honest finding the gates exist to resolve).

## Rejected alternatives

- **Prose-only validation plan:** not regressable; coverage rots silently.
  The gate registry is data with tests. Rejected.
- **Perturbing /sim defaults in place:** couples studies to global state and
  risks polluting other phases' determinism. Copies only. Rejected.
- **Intercept count / leaker count as headline metric:** nearly flat across
  the envelope (pursuit is truth-space); would have reported false
  robustness. Rejected for margin.
- **Full Monte Carlo over the envelope now:** belongs to PX, which owns the
  evaluation loop and headline figure. P-S2R deliberately ships the smallest
  harness that ranks the gates. Deferred.

## Acceptance evidence

- 36/36 tests pass in `sim/tests/test_ps_s2r.py` (232 cumulative, 1 skip).
- Gate coverage complete: `coverage_check() == []` (criterion A).
- Probe determinism: same gap + seed → identical event-log SHA-256 (B).
- ≥3 dimensions show material (−10 m+) margin loss → tornado genuinely
  prioritises the campaign (C).
- Insensitive dims each carry a gate; scope grep clean (D).
