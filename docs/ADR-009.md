# ADR-009: P-C2 — human-on-the-loop console as an architectural property

**Status:** Accepted  
**Phase:** PV — P-C2, human-on-the-loop C2 console

## Context

AEGISNET is a lethal/near-lethal engagement system. Even in simulation, the
architecture must reflect the operational requirement: no terminal engagement
happens without human authorisation. This is not a UI feature — it is a
safety invariant verifiable by code inspection and tested mechanically.

The C2 console is also the integration point for all five completed pillars:
fusion tracks (P2), classification confidence and rationale (P3), swarm intent
and value multipliers (PB), magazine state and allocation decisions (PA), and
the comms mesh topology (PC).

## Decisions

### Authorization interlock as a code-level invariant

`C2State.can_engage(track_id)` is the sole predicate that gates calls to
`world.assign()`. The call path is:

```
allocator.allocate() → ASSIGN recommendations
  → for each ASSIGN: c2_state.can_engage(track_id) must be True
    → world.assign(interceptor_id, entity_id)
```

No other code path calls `world.assign()` with a non-None target from a
fusion-derived ASSIGN decision. This makes the HOTL invariant grep-verifiable:
`grep -n "world.assign" sim/bridge/scenario.py` shows exactly one call site,
guarded by `can_engage()`.

### C2State is the sole C2 authority

`C2State` is a pure data object that holds:
- `weapons_hold: bool` — global halt; overrides all per-track authorisations
- `authorized_tracks: Set[str]` — tracks the operator has approved for engagement
- `held_tracks: Set[str]` — operator-placed HOLD (prevents engagement even if allocator bids)
- `friendly_marked: Set[str]` — MARK FRIENDLY; caps label at "unknown" in classifier
- `lambda_cost: float` — cost-exchange knob forwarded to AllocInput each round

Only the C2 bridge (`sim/bridge/`) modifies C2State. The allocator, classifier,
and fusion pipeline never read or write it.

### WEAPONS HOLD is synchronous to the next tick boundary

When the operator sends `WEAPONS_HOLD: true`, the flag is set on C2State
immediately (thread-safe via asyncio). On the **next tick** (20 ms at 50 Hz),
`can_engage()` returns False for every track. The world executes kinematics
normally — interceptors already in flight continue their trajectories — but no
new interceptor target is set. The acceptance test asserts this within one tick.

### Air picture: track uncertainty as a first-class display element

Tracks are rendered with 1-sigma Kalman covariance uncertainty ellipses.
Classification colour: hostile=red, unknown=amber, friendly=green; opacity
proportional to confidence. Intent-classified clusters are ringed in doctrine
colour: crimson=main_axis, orange=feint, cyan=isr_loiter, grey=reserve/unknown.
Assignment lines from interceptor to track: green when authorized, amber when
the allocator has bid but authorization is pending, grey for HOLD_FIRE.

### Audit trail completeness

Every C2 decision appends an audit event:
- `AUTHORIZED_ENGAGE` — interceptor+track pair forwarded to world.assign()
- `HOLD_PENDING_AUTH` — allocator bid, authorization not yet granted
- `WEAPONS_HOLD_ACTIVE` — bid rejected by global weapons hold
- `OPERATOR_HOLD` — bid rejected by per-track operator hold
- `MARK_FRIENDLY` — track reclassified as friendly by operator
- `AUTHORIZE` / `LIFT_HOLD` — operator state changes

The audit trail is append-only and scrollable in the console. It is also
replicated into the JSONL event log, making it part of the determinism hash.

### Replay scrubber

The console can load a JSONL event log and replay it at 1x/2x/4x speed.
The air picture reconstructs from the event stream without re-running physics.
Replay determinism: `log_hash(replay_log) == log_hash(original_log)`.

### ROE editor (lambda dial + value multiplier weights)

The operator can adjust `lambda_cost` (0–1 slider) live. Changes are forwarded
to the bridge and reflected in the next allocation cycle. The current λ value
is shown next to each HOLD_FIRE provenance entry in the audit trail.

## Acceptance evidence

- 14/14 Python tests pass (test_pv_c2.py).
- WEAPONS HOLD: after global hold is set, zero world.assign() calls with
  action=ASSIGN in the following tick. Verified mechanically.
- Replay determinism: same seed produces identical JSONL hash across
  independent runs. Verified in test_pv_determinism.
- TypeScript console compiles (tsc --noEmit) and Playwright spec passes
  headlessly against the running bridge server.

## Rejected alternatives

- **Automatic authorisation mode ("fire-and-forget")**: rejected unconditionally.
  The HOTL invariant is non-negotiable. An automatic mode would require a
  separate safety argument outside this ADR's scope.
- **Per-interceptor authorisation granularity**: operator fatigue under swarm
  stress makes per-interceptor decisions intractable. Per-track (one decision
  per threat estimate) is the correct granularity. Rejected.
- **Embedding C2State in the allocator**: the allocator is a pure cost-exchange
  optimiser and should carry no operator state. Allocation and authorisation are
  distinct concerns. Rejected.
- **Polling the bridge server from the frontend**: the air picture needs <200 ms
  latency to be usable under swarm stress. Server-push WebSocket is the correct
  transport; polling at any practical rate introduces jitter that degrades the
  operator mental model. Rejected.
