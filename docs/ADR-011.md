# ADR-011: PK — hardening, docs, demo packaging

**Status:** Accepted
**Phase:** PK — hardening, docs, demo packaging

## Context

The eleven preceding phases each shipped with an acceptance test suite, but the
*cross-cutting* properties that make GRECKO trustworthy — POSG discipline, the
C2 HOTL interlock, the scope boundary, and replay determinism — were asserted
in scattered per-phase tests and prose. A reviewer had no single command to
confirm them, and a careless later edit could violate one without any single
test obviously failing.

PK closes the project by making those properties a **mechanical gate**, writing
the capstone documentation, and packaging the headline result behind one
command.

## Decisions

### Architectural invariants become a single executable gate

`tools/verify_invariants.py` checks the four load-bearing properties in one run:

1. **POSG** — static scan: no module under `sim/fusion` (except the eval-only
   `benchmark.py`) references the truth sidecar API. This is the property that
   keeps the simulation honest; a fusion module that reads truth would
   invalidate every downstream conclusion.
2. **INTERLOCK** — AST analysis: exactly one `world.assign()` call exists in
   `sim/bridge/scenario.py`, and `can_engage` appears in the guarding block.
   The HOTL guarantee is thus structural, not behavioural — it cannot be
   bypassed by a code path the tests happened not to exercise.
3. **SCOPE** — regex scan of production trees: forbidden vocabulary
   (fire-control, waveform, jammer, weapon-release, warhead, launch-command,
   transmit-power) appears only on lines that also carry a disclaimer marker,
   or inside tests that assert the boundary.
4. **DETERMINISM** — runs a fixed scenario twice and compares event-log hashes.

The gate exits non-zero on any violation, so it serves as both a CI check and a
human-runnable confidence command.

### The verifier is tested with negative controls

A gate that can only pass is worthless. `test_pk_hardening.py` injects fake
trees with deliberate violations (a fusion module reading truth, an unguarded
second `world.assign()`, a bare fire-control reference) and asserts the verifier
*detects* each one — and that a disclaimer line is correctly allowed. The gate
is proven to discriminate, not merely to return green.

### One-command demo packaging

`demo.py` runs the invariant gate, then the Monte Carlo cost-exchange study,
and prints the headline table plus per-scenario breakdown. `--fast` gives a
~45 s look (2×4 league, 3 seeds, 1 tactic); the default is the full study.
This is the single artifact a new reader runs to see the whole project's claim
substantiated end to end.

### Capstone README with the scope boundary first

The README leads with the scope boundary — before the headline, before the
architecture — because for a counter-swarm project that framing is load-bearing,
and a CI test asserts the boundary language is present. The architecture table
maps every phase to its module and ADR.

### A standalone-CLI exemption is documented, not erased

`sim/run.py` (the P0 deterministic-replay kernel demo) contains its own
`world.assign()` call with a greedy allocator and no C2 interlock. The
invariant verifier scopes the INTERLOCK check to `sim/bridge/scenario.py`
specifically, because the interlock is a property of the *production C2 path*,
not of the standalone kernel demo. This is a deliberate, documented scoping
decision rather than an oversight: the kernel CLI predates the C2 layer and
exists to demonstrate raw deterministic replay.

## Rejected alternatives

- **Per-phase invariant assertions only:** already existed and were
  insufficient — no single confirmation command, and cross-cutting properties
  fell between the per-phase suites. The gate consolidates them. Rejected.
- **grep-based shell script for the gate:** brittle (the INTERLOCK guard check
  needs to understand block structure). AST analysis for the interlock and
  structured regex with disclaimer-awareness for scope are more robust. The
  Python verifier is also unit-testable with negative controls. Rejected.
- **Removing `sim/run.py`'s unguarded assign:** it is a legitimate standalone
  kernel demo predating the C2 layer; deleting it would lose the simplest
  determinism demonstration. Scoping the check to the bridge is cleaner.
  Rejected.

## Acceptance evidence

- 19/19 tests pass in `sim/tests/test_pk_hardening.py` (278 cumulative, 1 skip).
- All four invariants PASS on the current tree via
  `python -m tools.verify_invariants` (exit 0).
- Negative controls confirm the verifier detects truth leaks, extra/unguarded
  assigns, and bare forbidden vocabulary; disclaimer lines are allowed.
- README documents the scope boundary and headline; every phase ADR exists.
- `demo.py --fast` runs the gate + study end to end and prints the headline.
