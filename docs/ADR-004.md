# ADR-004: Market-based allocation vs centralized optimization — the economic magazine-rationing allocator

**Status:** Accepted  
**Phase:** PA — Pillar A, economic magazine-rationing allocator

## Context

Standard Weapon-Target Assignment (WTA) maximises kill probability on the
*current* picture. It has no concept of magazine depth, future waves, or
cost asymmetry between interceptor and threat. A $90k kinetic round against a
$500 decoy drone is a bad trade even if Pk = 0.85. This allocator formalises
the *sequential, magazine-constrained, cost-aware* problem and can output
HOLD_FIRE on purpose — something no textbook WTA can do.

## Formalism (encoded in economic_mdp.py and this ADR)

**State:** confirmed tracks (with intent distribution from Pillar B, using uniform
prior until PB is wired), remaining magazine by effector type, asset value,
estimated waves remaining.

**Objective** (rolling horizon H):

```
minimise  E[ Σ_j v_j * P(leak_j) ]  +  λ * E[ magazine cost spent ]
s.t.      per-type magazine budgets, reload/range/Pk feasibility.
```

Where:
- `v_j` = `priority_score` from P3 (TTA-dominant confidence-weighted value)
- `P(leak_j)` = 1 − Pk(effector, threat_type) if assigned, else 1
- `λ` = `lambda_cost` (dimensionless knob; `benefit = v_j*Pk − λ*(cost/asset_value)`)

**The λ normalisation fix:** Earlier versions used `cost_usd` directly, making λ
scale-dependent (λ=1 meant $90k rounds were worthless against 0.1-priority targets).
After debugging the decoy scenario, costs are normalised by `asset_value` so λ is
a clean [0,1] knob: λ=0 engage everything, λ=1 only engage if kill_value exceeds
cost fraction of the asset. This is documented here because it was a real design
error found during per-phase testing.

## Three solvers behind one `Allocator` interface

| Solver | Role | Comms constraint |
|---|---|---|
| `GreedyMyopic` | Baseline / current best practice | Global (respects adjacency but uses full picture) |
| `EconomicMDP` | **The pitch** — receding-horizon cost-exchange | Per-partition: each comms cluster optimises independently |
| `OracleLP` | Eval upper bound; full-info, no comms constraint | None — omniscient |

`OracleLP` is **eval only**: never the deployed path. Its existence as a
separate, clearly labelled class enforces this.

## EconomicMDP implementation

Receding-horizon greedy with cost-exchange gating and per-partition Hungarian
assignment. Exact MDP value-function computation is O(2^n·n) — deferred to v2
with RL policies from the league. This version solves the single-horizon problem
exactly: compute the full benefit matrix, run Hungarian on the benefit, gate out
any assignment with net benefit ≤ 0 (HOLD_FIRE), log the reason.

Decentralisation: interceptors are grouped into comms-partition-connected
components. Each partition sees only its own interceptors and the full track
picture visible to it. When the mesh fragments, duplicate assignments across
partitions are structurally possible (two isolated interceptors both engaging
the same target) — but within each partition there are no duplicates. This
matches real-world graceful degradation.

## Provenance logging

Every Assignment carries: `solver`, `bid_value`, `track_value_estimate`,
`magazine_state` at decision time, `round` index, and (for HOLDs) `hold_reason`.
This is the audit trail the HOTL console (PV) displays. Every HOLD_FIRE can
answer: "who declined this engagement, what was the estimated value, what was
the magazine state, and why."

## Rejected alternatives

- **Pure CBBA (Consensus-Based Bundle Algorithm):** CBBA handles the
  consensus/convergence property well but has no native magazine-rationing
  mechanism. The plan listed it as one of three allocators; after analysis,
  EconomicMDP (which uses Hungarian *inside* each partition step) provides the
  key novel behaviour (deliberate HOLD_FIRE) more directly. CBBA's distributed
  consensus property is partially recovered by the per-partition grouping. Full
  CBBA is deferred to v2 as the third allocator option.
- **Separate allocation and hold-fire stages:** holding is the output of the
  same benefit computation — splitting it into two stages adds latency and
  creates consistency bugs. Rejected.
- **Asset damage in dollar units for v_j:** cleaner for cost-exchange but
  requires an asset_value propagation chain through P3 and PB. The normalised λ
  achieves the same scale-independence without modifying the classification
  layer. Rejected for now; v_j can be dollar-denominated once PB produces
  wave-level damage estimates.

## Acceptance evidence

- Decoy scenario (λ=0.5, 8 kinetic rounds, 6 decoys + 6 main-axis targets):
  EconomicMDP assigns 0 decoys, 6+ main-axis; GreedyMyopic assigns by proximity.
- All three solvers: every interceptor appears exactly once, no duplicate targets.
- All HOLD_FIRE decisions have non-empty hold_reason (audit trail invariant).
- Cost-exchange ratio ≥ 0 for all solvers.
- Partition isolation: fully-partitioned mesh → each interceptor decides independently.
