# ADR-000: Fixed-timestep simulation + JSONL replay substrate

**Status:** Accepted  
**Phase:** P0 — Kernel & deterministic replay

## Context

The GRECKO simulation must satisfy two hard requirements simultaneously:
1. **Determinism** — identical seeds must produce identical outcomes across runs and machines, enabling reproducible Monte Carlo experiments and bit-exact regression tests.
2. **Replay & eval substrate** — every engagement must be replayable frame-by-frame by the C2 console and ingestible by the evaluation harness without replaying the physics.

## Decision

### Fixed timestep at 50 Hz (dt = 0.02 s)

All physics advancement occurs in equal-duration ticks. The timestep is the single most
important determinism lever: variable-timestep integrators accumulate floating-point
differences that compound differently on different hardware and load conditions.

50 Hz is fast enough to capture UAS kinematics faithfully (typical intercept geometries
close at ~10–70 m/s; 0.02 s gives 0.2–1.4 m position error per tick, well within the
15 m intercept radius) while keeping Monte Carlo cost manageable.

### Seeded RNG passed explicitly

A single `numpy.random.default_rng(seed)` instance is created at run start and threaded
explicitly through every stochastic function. No global random state is used. This is the
only other requirement for determinism given fixed-timestep integration.

### JSONL event stream as the single source of truth for replay/eval

Every discrete event (spawn, state change, intercept, leaker, asset damage) is appended
to an in-memory list and optionally streamed to a `.jsonl` file. The event log is the
*only* substrate that the C2 console, evaluation harness, and audit trail consume — no
shared mutable world state crosses layer boundaries.

The SHA-256 of the canonical JSONL string is the determinism acceptance criterion: two
runs with the same seed must produce the same hash.

## Rejected alternatives

### Variable timestep (e.g. Runge-Kutta with adaptive step)
Higher accuracy for smooth trajectories, but breaks determinism and makes the replay
substrate order-dependent. Rejected.

### Binary state snapshots instead of event log
Faster to write/read but opaque to downstream analysis. JSONL is human-readable,
grep-able, and directly schema-validated against `/proto/event.schema.json`.
Rejected.

### Per-entity random seeds
Easier to add entities without re-seeding everything, but breaks reproducibility when
entity spawn order changes. Single global RNG passed explicitly is simpler and
sufficient. Rejected.

## Consequences

- All downstream layers (sensing, fusion, allocation) must accept the world's RNG
  as a parameter — no module may instantiate its own.
- The 50 Hz timestep is a public contract: eval scenarios must specify `dt: 0.02`.
- The JSONL event log grows linearly with engagement duration; for the longest
  Monte Carlo scenarios (10-minute engagements, 100 entities) this is ~50 MB — acceptable.
